# 纽约时报畅销书排行榜前端展示需求总结

本文档仅聚焦纽约时报畅销书排行榜核心展示需求，剔除所有小红书、翻译相关内容，重点保障榜单数据可视化、信息完整性、操作便捷性及多端适配，贴合图书榜单的文艺属性与用户浏览习惯，明确前端各模块展示规范与交互逻辑。

# 一、核心布局与整体风格

1. **整体调性**：简约文艺、清晰大气，以浅色系（白色/米白色）为底色，搭配低饱和度辅助色（墨蓝、浅灰、淡绿），贴合图书内容的静谧质感，避免视觉杂乱。

2. **响应式布局**：全面支持PC端、移动端、平板端自适应，优先适配移动端竖屏浏览体验，PC端适配大屏展示，确保各终端信息布局合理、无错乱。

3. **页面结构**：采用「顶部导航+分类筛选区+榜单核心展示区+图书详情区」的层级结构，逻辑清晰，符合用户“筛选-浏览-查看详情”的核心操作路径，无多余冗余模块。

# 二、分类筛选区展示需求

1. **榜单分类展示**：严格按照纽约时报原版榜单分类展示，核心分类包括：精装小说（Hardcover Fiction）、精装非虚构（Hardcover Nonfiction）、平装小说（Trade Fiction Paperback）、平装非虚构（Paperback Nonfiction），分类名称以英文为主、中文注释为辅（标注在英文下方，浅灰色小字），支持分类标签一键切换，切换后无刷新加载对应榜单数据。

2. **筛选功能**：提供关键词搜索框，支持英文关键词搜索（书名、作者、出版社），具备输入联想、模糊匹配功能，搜索结果实时过滤榜单内容，无需刷新页面。

3. **状态提示**：分类切换、关键词搜索时，显示轻量加载动画（贴合图书主题，如翻书、书签动效）；无匹配结果时，显示友好提示（如“暂无匹配图书，请更换关键词重试”），搭配简约图书图标，提升体验。

4. **数据更新提示**：筛选区右上角展示榜单最新更新时间，字体偏小（浅灰色），格式为“最新更新：YYYY-MM-DD HH:MM”，明确告知用户数据时效性，增强内容权威性。

# 三、榜单核心展示区需求

## （一）列表视图（默认展示形式）

1. **单本图书卡片**：采用「左图右文」卡片式布局，卡片圆角设计（8-12px），hover时轻微上浮+淡阴影效果，提升交互质感；卡片间距均匀，适配不同终端排版。

2. **卡片核心信息**：严格展示纽约时报榜单原版信息，无额外冗余内容，布局如下：
        

    - 左侧：图书封面图（自适应裁剪，比例协调，无封面时显示默认图书占位图，支持懒加载，提升页面加载速度）；

    - 右侧：依次展示——榜单排名（红色角标/加粗数字，如“NO.1”，突出排名优先级）、书名（英文，加粗）、作者（英文）、上榜周数（如“Weeks on list: 12”）、出版社（英文，小字）、核心标签（榜单分类标签，如“Hardcover Fiction”，1-2个即可）。

3. **榜单排序**：严格按照纽约时报原版排名，从高到低依次展示；排名有变化时，可增加简易标识（如“New”表示新晋上榜、“↑3”表示排名上升3位），标识颜色与排名颜色一致，不突兀。

4. **批量操作**：展示区顶部提供「批量选择+导出CSV」按钮，支持勾选多本图书，导出含榜单所有核心信息的表格（适配运营批量处理需求），按钮样式简约，不占用核心展示空间。

    - 可选：新增“详细描述（Detailed Info）”标签页，展示图书更多原版细节（如ISBN、内容目录片段），满足用户深度了解需求。

# 五、数据加载与异常展示需求

1. **加载状态**：所有数据请求（分类切换、搜索、详情打开、表格导出）均需显示轻量加载动画，动画风格贴合图书主题（如翻书、文字跳动、书签旋转），避免用户感知卡顿；加载动画时长控制合理，加载完成后平滑过渡，无突兀跳转。

2. **空状态**：榜单无数据、搜索无结果、数据加载失败时，显示友好空状态提示，包含简约图标+清晰文字，同时给出解决方案（如“数据加载失败，请点击刷新重试”“暂无匹配图书，换个关键词试试”），避免用户迷茫。

3. **异常降级**：榜单数据请求失败时，前端自动降级展示本地缓存的历史榜单数据（若有），并在页面角落显示轻微异常提示（浅灰色小字，如“当前为历史缓存数据”），不影响核心浏览体验；无缓存数据时，显示失败提示并提供“刷新”按钮。

4. **缓存提示**：若展示的为缓存数据，在页面底部显示小字提示（如“本次展示为历史缓存数据，点击刷新获取最新榜单”），支持用户手动刷新数据，刷新后更新展示内容与更新时间。

4. **无干扰设计**：页面无任何无关广告、冗余模块，仅展示纽约时报畅销书排行榜相关信息与核心操作按钮，保持页面简洁，聚焦榜单本身。

5. **兼容性**：兼容主流浏览器（Chrome、Edge、Safari、Firefox）与移动端系统（iOS、Android），最低适配iOS 13、Android 10，确保不同设备用户均能正常浏览、操作。
