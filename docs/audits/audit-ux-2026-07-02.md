**审计日期**：2026-07-02

## 执行摘要

本次 UX 审计聚焦移动端用户在使用过程中最直接感受到的加载、导航、可达性、主题一致性与无障碍体验。整体来看，BookRank 移动端视觉风格统一、信息层级清晰、底部 Tab 导航明确，但在**加载反馈、无障碍焦点管理、暗色主题、搜索可达性、错误页一致性**方面仍存在明显体验缺口。

**综合裁定：CONCERNS（需改进）**

主要问题集中在：

1. 首页及分类切换缺乏骨架屏或加载指示，慢网络下容易出现白屏。
2. 移动端缺少暗色主题实现，与桌面端能力不一致。
3. 焦点管理、ARIA 状态与键盘操作支持不足，影响无障碍用户。
4. 搜索入口在移动端首页不可见，用户需要记忆 `/search` 路径。
5. 语言切换后仅依赖整页刷新，缺少加载反馈与读屏通知。
6. 桌面端与移动端错误页视觉风格、按钮布局不完全统一。

## 详细说明

### 1. 首页 loading / 骨架屏分析

`templates/mobile/index.html` 在书籍列表区域直接根据 `books` 变量渲染内容（line 49-94）。当服务端需要调用 NYT / Google Books API 时，用户从点击到首字节返回之间面对的是空白页面；在 3G/慢 Wi-Fi 环境下感知尤为明显。

- 模板中不存在 `.skeleton`、`.loading`、`.spinner` 等占位类。
- `static/mobile/css/mobile.css` 中没有任何骨架屏动画定义。
- `static/mobile/js/mobile.js` 没有监听页面加载进度或拦截导航以显示加载层。
- 仅有 `m-empty` 空状态（line 86-93）用于数据返回后无书场景，不能覆盖“等待中”场景。

**影响**：首屏等待时间长、用户不确定页面是否在响应、分类切换/搜索提交时体验断裂。

### 2. 分类切换、搜索、语言切换体验分析

#### 2.1 分类切换

`templates/mobile/index.html:22-31` 使用横向滚动的 `<a>` 链接实现分类切换，每次点击都会触发整页刷新：

```html
<a href="/?category={{ key }}"
   class="m-cat-tab {% if key == current_category %}active{% endif %}">
    {{ label }}
</a>
```

- 当前激活分类仅通过 `.active` 样式体现，缺少 `aria-current="page"`。
- 点击后到下一页返回前没有加载指示，用户可能重复点击。
- 未使用 `prefetch`/`preload` 提前准备下一分类数据。

#### 2.2 搜索

`templates/mobile/search.html:18-26` 提供完整的搜索表单，但在 `index.html` 与底部导航中均无搜索入口。移动端用户只有在知道 `/search` 路径或从其他页面进入时才能搜索。`app/routes/main.py:656` 返回 `active_tab='search'`，而 `templates/mobile/base.html` 底部导航没有搜索 Tab，导致搜索页没有任何 Tab 被高亮。

#### 2.3 语言切换

`templates/mobile/base.html:47-54` 在页面右上角固定放置地球图标按钮，下拉菜单包含中英文选项。`static/mobile/js/mobile.js:104-144` 中 `switchLanguage` 直接跳转 `/set-language?lang=...`：

```javascript
function switchLanguage(lang) {
    lang = lang === 'en' ? 'en' : 'zh';
    setSavedLanguage(lang);
    const next = window.location.pathname + window.location.search + window.location.hash;
    window.location.href = '/set-language?lang=' + lang + '&next=' + encodeURIComponent(next);
}
```

- 跳转期间页面冻结，没有加载提示。
- `m-toast-container`（`aria-live="polite"`）未用于播报语言变更。
- 打开下拉后按 `Esc` 不会关闭；焦点不会自动回到地球按钮。

### 3. 错误页一致性分析

桌面端 `templates/error.html` 使用项目公共样式类（`btn btn-primary`、`btn btn-secondary`）与 SVG Sprite，整体视觉偏横向布局；移动端 `templates/mobile/error.html` 使用独立样式 `.m-error-page`、独立内联 SVG，按钮纵向堆叠。

| 维度 | 桌面端 | 移动端 |
|------|--------|--------|
| 标题 | `{{ _('出错了') }} - BookRank` | `{{ _('出错') }} - 书榜` |
| 返回按钮 | 使用 `back_url` | 使用 `back_url`（可选） |
| 错误图标 | `#icon-x-circle`，色 `#e74c3c` | 内联 SVG，色 `--color-state-error #C0392B` |
| 按钮布局 | 横向 flex row | 纵向 flex column |
| 元信息 | 无 og/tw title | 有 `og_title`/`tw_title` |

虽然两端分别看都可用，但跨端品牌一致性不足，维护两套样式也增加了后续更新成本。

### 4. 移动端 bottom nav 与可达性分析

`templates/mobile/base.html:58-91` 定义了底部 4 个 Tab：首页、获奖书单、出版社、我的。

- 每个 Tab 使用 `<a>` + SVG + `<span>` 文本，语义清晰，但未设置 `aria-current="page"`。
- 当前激活状态通过 `.active` 类与填充 SVG 表现，对色盲用户仅依赖颜色/填充变化，不够稳健。
- Tab 点击区域 `min-height: var(--tab-height)`（50px），满足 44×44 触控目标（WCAG 2.5.5）。
- 缺少“跳过导航”链接（skip link），桌面端 `templates/base.html:75` 已有。
- 搜索作为高频功能未出现在底部导航，可达性受限。

### 5. 暗色主题 / 对比度分析

`static/mobile/css/mobile.css` 仅定义了一套浅色 Design Token（line 5-96），没有针对 `prefers-color-scheme: dark` 或 `[data-theme="dark"]` 的覆盖。桌面端 `static/css/base.css:101-169` 已经完整支持系统暗色与手动切换，移动端能力落后。

对比度检查：

- `--color-text-primary #333333` 在 `--color-bg #FAF6F0` 上对比度约 11.5:1，优秀。
- `--color-text-secondary #666666` 在 `--color-bg #FAF6F0` 上对比度约 5.5:1，满足 AA。
- `--color-accent #E67E22` 在 `--color-bg-elevated #FFFFFF` 上作为按钮背景，与白色文字对比度约 3.0:1；对于 14px 常规字重文字，未达到 WCAG AA 的 4.5:1 要求。

### 6. 无障碍（ARIA label、焦点管理）分析

#### 6.1 ARIA label

- 地球按钮、搜索输入框、返回按钮、取消收藏按钮等已有 `aria-label`，覆盖较好。
- 底部导航链接依赖可见文本标签，符合要求，但缺少 `aria-current="page"`。
- 分类 Tab 使用 `<section aria-label="图书分类">`，但内部链接未标注当前项。

#### 6.2 焦点管理

- `static/mobile/css/mobile.css:325-333` 中搜索输入框设置 `outline: none`，全局未补充 `:focus-visible` 样式，键盘用户难以识别焦点位置。
- `.m-tab`、`.m-cat-tab`、`.m-lang-globe-btn`、`.m-header-btn` 等交互元素均无可见焦点环。
- 语言下拉打开后，`Tab` 会离开下拉菜单进入页面其他区域；按 `Esc` 不会关闭下拉。

#### 6.3 读屏通知

- `m-toast-container` 已设置 `aria-live="polite"`，但语言切换、搜索提交等关键操作未向该区域写入状态消息。

## 优先级排序的 UX 改进建议

### P0 — 为首页及分类切换增加骨架屏与加载反馈

**优先级理由**：直接解决用户首次进入和切换分类时的白屏焦虑，投入小、感知强，且为后续客户端无刷新切换打好基础。

**落地方案（wireframe）**：

在 `templates/mobile/index.html` 的书籍列表区域之前插入骨架屏占位，服务端渲染真实内容后由 JS 隐藏。同时，在分类 Tab 点击与搜索提交时显示全局加载层。

```html
<!-- 首页骨架屏占位（index.html 中 books 列表外层） -->
<div class="m-skeleton-list" id="m-skeleton" aria-hidden="true">
  {% for _ in range(5) %}
  <div class="m-skeleton-card">
    <div class="m-skeleton-rank"></div>
    <div class="m-skeleton-cover"></div>
    <div class="m-skeleton-body">
      <div class="m-skeleton-title"></div>
      <div class="m-skeleton-line"></div>
      <div class="m-skeleton-line m-skeleton-line--short"></div>
    </div>
  </div>
  {% endfor %}
</div>
```

- 在 `mobile.css` 中增加全局 `:focus-visible` 焦点环：

```css
:focus-visible {
    outline: 2px solid var(--color-primary);
    outline-offset: 2px;
}
```

- 为底部导航链接增加 `aria-current="{% if active_tab == 'home' %}page{% endif %}"`。
- 为分类 Tab 当前项增加 `aria-current="page"`。
- 为 `#m-lang-dropdown` 增加 `aria-controls` 与 `Escape` 关闭，并在关闭后焦点回到触发按钮。
- 在 `templates/mobile/base.html` `<body>` 开头增加 skip link：

```html
<a href="#main-content" class="m-skip-link">{{ _('跳转到主要内容') }}</a>
```

并为主内容容器补充 `id="main-content"`。
