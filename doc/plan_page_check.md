## 二、检查标准

### 2.1 翻译质量检查
- **书名**：应为简洁中文书名，格式正确（有《》或纯中文）
- **禁止出现的污染模式**：
  - markdown标记（`**`、`##`、`_`、`` ` ``）
  - 字段标签（"书名："、"作者："、"简介："、"Title:"、"Author:"）
  - 间隔号+作者名混入（如"丹尼尔·卡尼曼译"）
  - 书名后有长描述（如"《掌控习惯》詹姆斯·克利尔 这是一本..."）
  - 多行长文本（换行符+长度>30）
  - 双书名号（"《《"）
  - 末尾"译"字且长度>2

### 2.2 功能检查
- 页面HTTP状态码为200
- 无JavaScript控制台错误（CSP、404、undefined等）
- 图片加载正常（封面图、默认图fallback）
- 交互元素正常（按钮、链接、展开/收起）

### 2.3 数据一致性检查
- 书名与作者名分开显示，不混在title中
- 描述内容格式正确
- 分类名称正确显示

## 三、执行步骤

### 阶段1：首页分类批量检查
1. 依次访问8个分类页面
2. 每个页面提取前15本书的书名
3. 逐条检查是否符合翻译质量标准
4. 记录问题书名、ISBN、期望修复方式

### 阶段2：功能页面检查
1. 访问 `/awards`、`/new-books`、`/publishers`
2. 检查页面是否加载、数据是否正常
3. 检查控制台是否有错误

### 阶段3：周报页面检查
1. 访问 `/reports/weekly`
2. 点击最新周报进入详情
3. 检查书名是否有双书名号等格式问题

### 阶段4：详情页抽样
1. 从首页点击一本书进入详情
2. 检查标题、描述、作者等字段翻译

### 阶段5：问题修复
1. 对发现的问题分类（前端/后端/数据）
2. 前端问题：修改JS/HTML
3. 后端问题：修改Python逻辑
4. 数据问题：使用admin API清理或更新数据库

### 阶段6：回归验证
1. 修复后重新访问问题页面
2. 确认问题已解决
3. 更新CHANGELOG.md

## 四、问题分级

| 级别 | 定义 | 处理方式 |
|------|------|----------|
| P0 | 页面无法加载/功能完全不可用 | 立即修复 |
| P1 | 翻译明显错误/污染严重 | 本批次修复 |
| P2 | 格式小问题/显示不美观 | 记录待后续优化 |
| P3 | 控制台警告/非功能性问题 | 记录待后续优化 |

1. 问题清单（含页面、问题描述、截图/证据）
2. 修复后的代码变更
3. 更新后的CHANGELOG.md
4. 回归验证报告
